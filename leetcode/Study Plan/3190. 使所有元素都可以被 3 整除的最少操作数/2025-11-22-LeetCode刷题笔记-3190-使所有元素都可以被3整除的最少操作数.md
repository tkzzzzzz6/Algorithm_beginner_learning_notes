# 题目信息
- 平台：LeetCode
- 题目：3190. 使所有元素都可以被 3 整除的最少操作数
- 难度：Easy
- 题目链接：[Make All Elements Divisible by Three](https://leetcode.cn/problems/make-all-elements-divisible-by-three/description/)

---

## 题目描述
> 给定一个整数数组 `nums`，每次操作可以任选一个下标，将 `nums[i]` 增加 1。问最少需要多少次操作，才能让数组中所有元素都能被 3 整除。

---

## 初步思路
1. 把元素拆成三类：余 0、余 1、余 2。
2. 对于余 0 的元素无需处理。
3. 对于余 1 或余 2 的元素，只需要加 2 或加 1 次即可让它变成 3 的倍数。
4. 因此每个“非 3 的倍数”的元素最少都要一次独立操作（逐个累积即可）。

---

## 算法分析
- 核心：计数元素 `nums[i] % 3 != 0` 的个数。
- 关键技巧：无需真的模拟加法；因为题目允许逐个元素独立地累积加 1。
- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

---

## 代码实现（Python）
```python
class Solution:
    def minimumOperations(self, nums: List[int]) -> int:
        total = sum(1 for x in nums if x % 3 != 0)
        return total
```

---

## 测试用例
| 输入 | 输出 | 说明 |
|------|------|------|
| nums = [1,2,3,4] | 2 | 元素 1、2 需要分别加 2 次和 1 次，但每个单独统计操作数即 2 |
| nums = [3,6,9] | 0 | 已全部可被 3 整除 |
| nums = [2,2,2] | 3 | 每个元素都需加 1 |

---

## 总结与反思
1. 题目本质是一个“按元素独立修正”的计数问题。
2. 若题目改成“一次操作可改多个元素”，才需要复杂策略；当前版本直接计数即可。
3. 注意 Python 列表解析即可快速完成统计。
