# D. 排行榜生成

## 题目描述


哈尔滨华德学院第十六届程序设计竞赛即将拉开帷幕！本次比赛共有 $$ 名选手， $$ 道题目，采用 $$ 单人赛制。  

每名选手的最终成绩由**解题数量**和**总罚时**决定。罚时计算规则如下：  
- 
若一道题未通过，罚时为 $$ ；  

- 若一道题通过，罚时为该题被通过的时间（单位：分钟）。  

- 注::本题为了简化，直接给出了一道 $$ 后的总罚时，如果罚时等于 $$ ，就是没通过。  


排名规则：  
- 
**解题数量多**的选手排名靠前；  

- 
解题数量相同时，**总罚时少**的选手排名靠前；  

- 
当多名选手解题数和总罚时完全相同时，他们获得**相同的排名**，并且**学号小的在前**（即输出时按学号升序排列），后续选手的名次跳过并列的人数。  

现在，你需要根据输入的选手信息和每道题的罚时，输出最终的排行榜。  

  

  
****注意：本题默认没有通过题目的人，也发生了提交，**没有成功解题的选手，在榜单上，排名统一是最后一名，（没有通过题目的人的排名并列为，排名最靠后的通过题目的人的排名加一，如果没有人通过题目 那么所有提交题目的人，并列第一。）。**  

  
  


## 输入描述


第一行两个整数 $$ ，分别表示选手人数和题目数量。  

接下来 $$ 行，每行首先是一个字符串 $$ （由10位数字组成，学号两两不同），表示选手学号，随后有 $$ 个整数 $$ ，表示该选手每道题的罚时（若为 $$ 表示该题未通过）。  
  


## 输出描述


输出 $$ 行，每行包含：**排名、学号、解题数、总罚时**，中间用一个空格隔开。  
若多名选手解题数和总罚时完全相同，则他们获得相同的排名，并且学号小的优先输出；后续选手的排名依次后移（例如：两人并列第 $$ ，则下一人排名为第 $$ ）。  
注意：是排行榜不是获奖名单，本题默认没有通过题目的人，也发生了提交，没有通过题目的人也在排行榜上。名次是最后一名，通过题目数和罚时都是 $$ 。  
  


## 样例

### 样例 1
**输入**:
```
3 4
2024000001 120 0 45 0
2024000002 90 60 0 30
2024000003 150 0 0 0
```

**输出**:
```
1 2024000002 3 180
2 2024000001 2 165
3 2024000003 1 150
```

### 样例 2
**输入**:
```
3 2
2024000001 50 60
2024000002 50 60
2024000003 70 0
```

**输出**:
```
1 2024000001 2 110
1 2024000002 2 110
3 2024000003 1 70
```
