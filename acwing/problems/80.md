# 思路 1:DFS
存在重复计算,会超时

```cpp
class Solution {
   public:
    vector<int> numberOfDice(int n) {
        vector<int> res;
        for (int i = n; i <= 6 * n; ++i)
            res.push_back(dfs(n, i));
        return res;
    }

    int dfs(int n, int sum) {
        if (sum < 0)
            return 0;
        if (n == 0)
            return !sum; // 如果 sum = 0返回 1,sum != 0返回 0

        int ans = 0;
        for (int i = 1; i <= 6; ++i) {
            ans += dfs(n - 1, sum - i);
        }

        return ans;
    }
};
```

# 思路 2:动态规划
## 状态定义：
设 $f[i][j]$ 表示用 $i$ 个骰子，掷出点数总和为 $j$ 的方案数。

## 边界条件：
$f[0][0] = 1$
表示“0 个骰子凑出 0 分”有 1 种方法，也就是不投掷。

其余初始状态都可以看成 $0$，尤其是：
- $f[0][j] = 0 \quad (j > 0)$
- $f[i][j] = 0$ 也适用于不可能的状态，比如 $j < i$ 或 $j > 6i$

## 状态转移方程：
最后一个骰子可以掷出 $1 \sim 6$，所以

$$
f[i][j] = \sum_{k=1}^{6} f[i-1][j-k]
$$

如果写得更严谨一点，就是只累加合法范围：

$$
f[i][j] = \sum_{k=1}^{\min(6, j)} f[i-1][j-k]
$$

含义是：
- 先固定第 $i$ 个骰子掷出的点数是 $k$
- 那么前 $i-1$ 个骰子必须凑出 $j-k$
- 把这 6 种情况加起来，就是 $f[i][j]$

对应到代码里的三层循环：
- $i$：枚举骰子个数
- $j$：枚举总点数
- $k$：枚举最后一次投掷的点数

```cpp
class Solution {
   public:
    vector<int> numberOfDice(int n) {
        vector<vector<int>> f(n + 1, vector<int>(6 * n + 1));
        f[0][0] = 1;

        for (int i = 1; i <= n; ++i) {                 // 枚举投掷次数
            for (int j = 1; j <= 6 * i; ++j) {         // 枚举点数总和
                for (int k = 1; k <= min(j, 6); ++k) { // 枚举最后一次投掷
                    f[i][j] += f[i - 1][j - k];
                }
            }
        }

        vector<int> res;
        for (int i = n; i <= 6 * n; ++i)
            res.push_back(f[n][i]);
        return res;
    }
};
```
